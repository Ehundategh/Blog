# 平衡树

二叉搜索树利用关键字的大小关系组织节点，可以在树高范围内完成查找、插入、删除、排名、前驱与后继等操作。但普通二叉搜索树的形态完全由插入顺序决定，最坏情况下会退化成一条链。

平衡树在二叉搜索树的基础上加入额外的结构约束，使树高保持在对数级别，或者保证每次操作的均摊复杂度为对数级别。

## 目录

- [二叉搜索树](binary-search-tree.md)：二叉搜索树的性质、节点维护、旋转与基本操作。
- [Splay](splay.md)：通过伸展操作得到均摊平衡。
- [Treap](treap.md)：利用随机优先级与堆性质得到期望平衡。

后两者都不会改变二叉搜索树的中序遍历，因此查找、排名、前驱与后继的基本逻辑均与普通二叉搜索树相同。
